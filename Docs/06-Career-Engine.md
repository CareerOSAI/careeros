# Purpose

Career Engine reprezintă motorul principal al platformei CareerOS.

Acest document definește modul în care platforma transformă un obiectiv profesional într-un plan personalizat și adaptiv.

Toate funcționalitățile produsului trebuie să utilizeze această logică.

---

# Philosophy

CareerOS nu oferă informații.

CareerOS generează progres.

Progresul este obținut printr-o succesiune de pași mici, măsurabili și adaptați utilizatorului.

# Definitions

## Goal

Obiectivul final al utilizatorului.

Exemple:

- Backend Developer
- Data Analyst
- Product Manager
- UX Designer

---

## Assessment

Analiza inițială.

CareerOS colectează informații despre utilizator:

- experiență
- studii
- timp disponibil
- obiective
- competențe
- interese
- limba
- locație

---

## Career Blueprint

Planul complet.

Este strategia care îl duce pe utilizator la obiectiv.

Blueprint-ul este unic pentru fiecare utilizator.

---

## Milestone

O etapă importantă.

Exemple:

- Învață Git

- Învață Python Basics

- Creează primul proiect

- Publică proiectul

- Simulare interviu

---

## Task

Cea mai mică unitate de lucru.

Exemple:

- citește capitolul 1

- rezolvă 5 exerciții

- creează repository

- implementează API

Task-urile trebuie să poată fi terminate într-o singură sesiune.

---

# Career Engine Rules

Rule 1

Un Goal poate avea mai multe Blueprint-uri.

---

Rule 2

Un Blueprint conține mai multe Milestones.

---

Rule 3

Un Milestone conține mai multe Tasks.

---

Rule 4

Task-urile trebuie executate într-o ordine logică.

---

Rule 5

AI poate modifica Task-uri.

---

Rule 6

AI poate modifica Milestones.

---

Rule 7

Goal-ul NU se modifică decât dacă utilizatorul decide.

# Adaptation

CareerOS verifică periodic:

- progresul;

- dificultatea;

- timpul disponibil;

- feedback-ul utilizatorului;

- rezultatele.

Dacă este nevoie, planul este actualizat.

---

# End Condition

Motorul își încheie execuția atunci când utilizatorul atinge Goal-ul.

În acel moment poate începe un Goal nou.
